## ๑. วัดได้อย่างไร โดยไม่ต้องนั่งพิมพ์เฉลยเอง

ชุดเอกสารที่ลูกค้าให้มามี**เอกสารเดียวกันสองรูปแบบ**

```
LandCode2497_Amend-v3_ImgPDF.pdf     สแกนจากราชกิจจานุเบกษา  → เป็นรูป ต้อง OCR
LandCode2497_Amend-v3_TruePDF.pdf    ไฟล์ตัวอักษรจากกฤษฎีกา  → ใช้เป็น "เฉลย"
```

คู่แบบนี้มี **๑๖ คู่** ในจำนวนนี้ **๙ คู่** ที่ฝั่งสแกนไม่มีตัวอักษรฝังอยู่เลย = ต้อง OCR จริง ๆ
เราจึงเอา ๙ คู่นี้เป็นชุดทดสอบ **ได้เฉลยมาฟรี ไม่ต้องจ้างคนพิมพ์**

### ⚠️ ข้อจำกัดที่ต้องรู้ก่อนอ่านตัวเลข

ทั้งสองไฟล์ **ไม่ได้เหมือนกัน ๑๐๐%** ฉบับราชกิจจานุเบกษามีหัวหนังสือ ตราครุฑ ลายเซ็น และหมายเหตุ
ที่ฉบับกฤษฎีกาไม่มี (จำนวนหน้าจึงต่างกัน เช่น Main-v0 มี ๖๗ หน้า แต่ฉบับเฉลยมี ๓๖ หน้า)

**แปลว่า:**
- ✅ **"อ่านเลขมาตราเจอ"** และ **"วลีสำคัญครบ"** — อ่านเป็นคะแนนดิบได้เลย
- ⚠️ **"ตัวอักษรตรงกับต้นฉบับ"** — ใช้เทียบระหว่าง OCR สองตัวเท่านั้น **ไม่มีทางได้ ๑๐๐% แม้ OCR จะถูกหมด**

## ๒. วัด ๔ อย่าง — เลือกตามความสำคัญต่อ "ระบบนี้"

ไม่ได้ใช้เกณฑ์วัด OCR ทั่วไป แต่เลือกวัดสิ่งที่**พังแล้วระบบเราเสียหายจริง**

| ลำดับ | วัดอะไร | ทำไมสำคัญ | ถ้าตกจะเกิดอะไร |
|---|---|---|---|
| **๑** | เลข "มาตรา ๙" อ่านเจอไหม | ระบบใช้เลขมาตราเป็นป้ายกำกับทุกชิ้นข้อมูล | **ติดป้ายผิด → ค้นยังไงก็ไม่เจอ** |
| **๒** | วลีที่ใช้ต่อสายการแก้ไข | เช่น "ให้ยกเลิกความใน", "แก้ไขเพิ่มเติมโดย" | **สายการแก้ไขขาด → ตอบ "ฉบับก่อนหน้าของก่อนหน้า" ไม่ได้** |
| **๓** | ตัวเลขไทย ๐-๙ | ปี พ.ศ. / จำนวนวัน / จำนวนปี | **ห้ามโอน ๕ ปี กับ ๑๐ ปี คือคนละกฎหมาย** |
| **๔** | สระ วรรณยุกต์ ตัวอักษร | ปัญหาคลาสสิกของ OCR ภาษาไทย | ค้นด้วยคำที่สะกดถูกแล้วไม่เจอ |

## ๔. ตัวอย่างความผิดพลาดจริง

จาก `Amend-v3` (พ.ร.บ.แก้ไขเพิ่มเติมฯ ฉบับที่ ๓ พ.ศ. ๒๕๒๖) หน้าแรก:

| | ข้อความ |
|---|---|
| **เฉลย** | แก้ไขเพิ่มเติมประมวลกฎหมาย**ที่ดิน** … พ.ศ. **๒๕๒๖** … ให้ไว้ ณ วันที่ **๑** ตุลาคม … **เป็นปีที่** ๓๘ |
| **OCR** | แก้ไขเพิ**มเติม**ประมวลกฎหมาย**ทีดิน** … พ.ศ. **๒&๒๖** … ให้ไว้ณ วันที่ **⟨หาย⟩** ตุลาคม … **เบ็นบีที่** ๓๘ |

สังเกต ๔ อาการ:
1. **ไม้เอกหลุด** — "เพิ่ม" → "เพิม" · "ที่ดิน" → "ทีดิน"
2. **ตัวเลขเพี้ยนเป็นสัญลักษณ์** — ๒๕๒๖ → ๒&๒๖
3. **ตัวเลขหายทั้งตัว** — วันที่ ๑ หายไปเฉย ๆ
4. **พยัญชนะสลับ** — "เป็นปีที่" → "เบ็นบีที่"

## ๕. ข้อสังเกตที่น่าสนใจ

**วลีที่เป็นแบบฟอร์มรอดดีกว่าตัวเลขมาก — ๘๑.๓% เทียบกับ ๔๔.๕%**

เพราะ OCR มีบริบทภาษาช่วยเดาคำได้ (เห็น "ให้ยกเลิกความใ_" ก็เดา "น" ถูก)
แต่**ตัวเลขไม่มีบริบทให้เดา** — ๕ กับ ๖ ในเลขไทยหน้าตาใกล้กัน เดาผิดก็ไม่มีอะไรมาค้าน

> นี่คือเหตุผลที่ **ตัวเลขคือจุดอันตรายที่สุดของ OCR ในงานกฎหมาย** และเป็นตัวชี้วัดที่ควรจับตาที่สุดตอนเทียบ OCR ตัวใหม่

## ๖. แล้วทำไมระบบยังตอบถูก ๒๘/๓๐ ทั้งที่ OCR แย่ขนาดนี้

เพราะออกแบบกันไว้ ๒ ชั้นตั้งแต่แรก:

**ชั้นที่ ๑ — ตัวบทที่ใช้บังคับปัจจุบันมาจากไฟล์ TruePDF ล้วน**
คำถามทั่วไปจะถูกล็อกให้ค้นเฉพาะกลุ่มนี้ ซึ่ง **ไม่มีข้อมูลจาก OCR ปนเลย**

**ชั้นที่ ๒ — ข้อมูลจาก OCR ถูกดันลงท้ายแถวเสมอ** (`_demote_scans`)
เพราะถือหลักว่า *"ถ้าไม่มั่นใจ อย่าเอาขึ้นก่อน"*

และ ๙ ไฟล์ที่ OCR อยู่นี้ **มีฉบับ TruePDF คู่กันครบทุกไฟล์** — ข้อมูลจาก OCR จึงแทบไม่ถูกใช้จริง

### 🎯 สรุปประเด็นนี้

> **ที่จะเปลี่ยน OCR ไม่ใช่เพราะตอนนี้พัง แต่เพราะตอนนี้เรา "ไม่ได้ใช้มัน"**
>
> วันไหนเจอเอกสารที่มีแต่ฉบับสแกน (ไม่มี TruePDF คู่) เราจะไม่มีตาข่ายรองรับ
> และถ้า OCR ดีพอ เราจะ**เลิกต้องดันข้อมูลสแกนลงล่าง** ได้ข้อมูลใช้งานเพิ่มอีก ๑๓๐ ชิ้น

| ตัวชี้วัด | ของเดิม | ตัวใหม่ต้องได้ | เหตุผล |

## ๘. วิธีรันซ้ำ / วัดตัวใหม่

```powershell
# วัดตัวเดิม (อ่านจาก cache — เสร็จในไม่กี่วินาที ไม่มีค่าใช้จ่าย)
.venv\Scripts\python.exe eval\ocr_bench.py --tag baseline

# วัดความเร็วจริง ต้องปิด cache ไม่งั้นได้ 0 วินาที
.venv\Scripts\python.exe eval\ocr_bench.py --no-cache

# วัด OCR ของบริษัท (ต้องตั้ง OCR_API_URL ใน .env ก่อน)
.venv\Scripts\python.exe eval\ocr_bench.py --engine company --no-cache

# ส่งผลขึ้น MLflow เพื่อเก็บถาวรและเทียบข้ามรอบ
.venv\Scripts\python.exe eval\ocr_bench.py --engine company --mlflow --tag company-ocr
```

**วิธีเปลี่ยนไปใช้ OCR ตัวใหม่จริง:** แก้ที่ `ocr.py` ฟังก์ชัน `page_text()` **จุดเดียว**
เพราะทั้งระบบเรียก OCR ผ่านฟังก์ชันนี้ฟังก์ชันเดียว ไม่ต้องแตะไฟล์อื่นเลย

## ๙. เรื่องความปลอดภัยที่ต้องพิจารณาก่อนเปลี่ยน

**EasyOCR ตัวเดิมรันในเครื่อง ๑๐๐%** — ตั้ง `download_enabled=False` ไว้ **ข้อมูลไม่ออกไปไหนเลย**

ถ้าเปลี่ยนเป็น API ของบริษัท เอกสารจะถูกส่งออกไปที่ service นั้น

- กรณีนี้เป็น **กฎหมายสาธารณะ** → ไม่มีปัญหา
- แต่ถ้าวันหลังเอาระบบนี้ไปใช้กับ **เอกสารภายในของบริษัทหรือลูกค้า** → ต้องทบทวนใหม่ก่อนใช้

⚠️ อย่าเขียน URL หรือ API key ลงในโค้ดโดยตรง — ให้ใส่ใน `.env` ซึ่งถูกกันไม่ให้ขึ้น git ไว้แล้ว
